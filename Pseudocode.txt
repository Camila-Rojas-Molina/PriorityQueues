/-----------------------------------------------------------------------------------
OUR PLAN
/-----------------------------------------------------------------------------------
- every entry lives in a min-heap and a max-heap at the same time
- each entry stores:
    - key
    - value
    - minIndex
    - maxIndex
the APQ stores:
    - minHeap
    - maxHeap
    - size
    - mode where mode is either MIN or MAX
Every time you swap two entries in the heap, you must also update their stored indices.

Design: A dual-heap APQ:
    one min-heap
    one max-heap
    each entry stores two indices: one in the min-heap and one in the max-heap
    toggle() just flips the current state, so it can be O(1)
We keep both views ready all the time.
We keep a min-heap and a max-heap at the same time, so switching is instant.
